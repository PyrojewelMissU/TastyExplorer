期末作品要求

1. 项目主题

  • 自行选题，完成一个能在手机浏览器正常使用的移动端 Web 应用。。
  • 作品可以是一个完整的应用程序，或者是一个特定功能的模块。（例如：一个具有交互功能的用户界面，或者一个数据可视化的应用。）

2. 技术栈

应使用以下技术来实现项目：

  • HTML5：负责网页结构的定义，包含语义化标签。
  • CSS3：美化页面，可以使用媒体查询等响应式设计技术，确保页面在不同设备上的表现。
  • JavaScript：核心逻辑实现，能够使用 ES6+ 新特性，如箭头函数、模板字符串、模块化等。
  • 前端框架或库（可选）：如 Vue.js、React 或 Angular，来构建单页应用（SPA）或组件化的界面。
  • API交互：通过 Axios 实现与外部 API 的数据交互，能够处理 JSON 数据。

3. 项目功能

项目应至少包括以下功能：

  • 基本的 UI 设计：界面设计清晰，能够体现用户交互逻辑。
  • 数据展示与交互：通过动态渲染页面展示数据，例如表单提交、表格、图表等。
  • 响应式设计：项目应兼容不同屏幕尺寸，如移动端、平板、桌面端。
  • 路由管理：如果是单页应用（SPA），需使用路由管理来处理不同的页面视图。
  • 错误处理与提示：合理地进行用户输入验证、错误处理，给出适当的提示信息。
  • API 调用与数据展示：可以通过模拟 API 数据（如天气数据、新闻数据等）或与后台 API 进行真实的交互。

4. 项目展示

  • 可用性与用户体验：项目应易于使用，并符合用户友好的交互设计。
  • 设计美学：页面的视觉效果和排版应简洁美观，颜色、字体搭配合理，尽量避免过于拥挤或混乱的布局。
  • 交互逻辑：实现一些动态效果（例如，按钮点击、表单提交、下拉菜单、模态框等）提升用户体验。

5. 项目文档

包含项目的背景、技术栈、安装和运行指南、功能介绍等内容。简单说明关键技术的使用，如何解决了一些常见的前端开发问题。

评分标准

1. 功能实现（30%）

  • 核心功能：项目是否完成了核心功能，是否符合要求的项目功能描述。
  • 数据交互：项目是否能够正确获取和展示外部数据（如通过 API 调用等方式）。
  • 响应式设计：项目是否兼容不同设备和屏幕尺寸。

2. 技术实现（30%）

  • 技术栈的合理性和应用：项目是否使用了适当的技术栈。
  • 代码结构与组织：代码是否模块化、清晰。
  • API 调用和错误处理：是否正确处理 API 数据、错误提示和输入验证。

4. 项目文档（40%）
