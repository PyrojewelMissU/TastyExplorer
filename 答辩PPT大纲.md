### 第 5 页：技术亮点 ① - 批量导入系统

**标题**: 四、技术亮点 ① 批量导入系统

**内容**:
```
❓ 问题
TheMealDB API 单次搜索只返回约 60 个菜品，无法满足需求

💡 解决方案
1. 使用 search.php?f={letter} 按首字母搜索
2. 并行请求 20 个字母（a-z）
3. 使用 Promise.all() 等待所有请求
4. Map 数据结构去重（以 idMeal 为 key）
5. 随机排序后取前 200 个

📊 核心代码
const letters = ['a', 'b', 'c', ..., 'w'];
const requests = letters.map(letter =>
  axios.get(\`/search.php?f=\${letter}\`)
);
const results = await Promise.all(requests);

// Map 去重
const allMeals = new Map();
results.forEach(meals => {
  meals.forEach(meal => {
    if (!allMeals.has(meal.idMeal)) {
      allMeals.set(meal.idMeal, meal);
    }
  });
});

✅ 效果
• 成功获取 150-250 个不重复菜品
• 请求时间 6-8 秒
• 一次加载，永久缓存
```

**演讲内容**（1.5 分钟）：
- 第一个技术亮点是批量导入系统
- API 限制每次只能返回 60 个菜品，我通过按首字母搜索突破了这个限制
- 并行请求 20 个字母，用 Promise.all 等待所有请求完成
- 用 Map 去重后可以获取 150 到 250 个不重复菜品
- 最终实现了一次加载 200 道菜品，并永久缓存到本地

export const translateText = async (text) => {
  // 检查缓存
  const cacheKey = \`en:zh:\${text}\`;
  if (translationCache.has(cacheKey)) {
    return translationCache.get(cacheKey);
  }

  try {
    // 尝试 MyMemory API
    let translated = await translateWithMyMemory(text);

    // 失败则尝试 LibreTranslate
    if (!translated) {
      translated = await translateWithLibre(text);
    }

    // 缓存结果
    if (translated) {
      translationCache.set(cacheKey, translated);
      return translated;
    }

    // 返回原文
    return text;
  } catch (error) {
    return text;
  }
};

### 可能的提问及回答

**Q1: 为什么选择 Vue 3 而不是 React？**
A: Vue 3 学习曲线平缓，Composition API 逻辑复用更好，而且轻量高效，适合这种中小型项目。

**Q2: 翻译 API 有频率限制怎么办？**
A: 我设计了三层策略：本地字典（70% 命中）、内存缓存、双重 API。大部分翻译不需要调用 API，即使 API 失败也会显示英文原文。

**Q3: 首次加载 6-8 秒是否太慢？**
A: 首次需要批量导入 200 道菜品，确实较慢。但导入后永久缓存，再次访问只需 0.32 秒。未来可以考虑分页加载或预加载优化。

**Q4: 有没有考虑过用 Vuex 管理状态？**
A: 项目状态相对简单，主要是收藏列表。用 localStorage + storage 事件就能实现跨组件同步，引入 Vuex 会增加复杂度。

**Q5: 如何保证应用的安全性？**
A: 应用是纯前端，没有用户敏感信息。API 调用使用 HTTPS，localStorage 存储的是公开数据。如果未来加入用户系统，会考虑 JWT 认证和数据加密。
